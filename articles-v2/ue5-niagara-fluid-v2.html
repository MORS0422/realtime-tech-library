Niagara 流体模拟：SPH与FLIP算法的实时GPU实现

Niagara 是 Unreal Engine 5 的模块化粒子系统，通过 GPU 模拟技术实现了影视级流体效果的游戏实时化。 本文从计算流体动力学（CFD）基础出发，深入解析 SPH（光滑粒子流体动力学）和 FLIP（流体隐式粒子）算法原理， 以及 Niagara 的 GPU 粒子管线架构，帮助开发者掌握实时流体模拟的核心技术。

🔬 核心突破

Niagara 流体模拟的核心在于GPU 并行计算架构。传统 CPU 流体模拟受限于串行计算， 只能处理数千个粒子。而 Niagara 利用 Compute Shader 在 GPU 上并行处理数十万个粒子， 通过空间哈希（Spatial Hash）加速邻域搜索，实现了游戏实时性能（60fps）下的高质量流体效果。

1. 概述与核心概念

1.1 计算流体动力学基础

计算流体动力学（CFD）是使用数值方法求解流体运动方程的学科。对于实时应用， 我们需要在物理准确性和计算性能之间找到平衡。Niagara 采用基于粒子的拉格朗日方法， 而非传统的基于网格的欧拉方法，更适合处理自由表面流体（如水、烟雾）。

📐 Navier-Stokes 方程

流体运动的数学描述基于 Navier-Stokes 方程： 

ρ(∂u/∂t + u·∇u) = -∇p + μ∇²u + f

其中：
- ρ: 流体密度
- u: 速度场
- p: 压力
- μ: 动力粘度
- f: 体积力（如重力）

1.2 SPH vs FLIP 方法对比

特性 SPH（光滑粒子流体动力学） FLIP（流体隐式粒子）
物理模型 纯粒子方法，无网格 粒子-网格混合方法
压缩性 弱压缩（需要状态方程） 不可压缩（投影法求解）
稳定性 需要较小时间步长 更稳定，允许较大时间步长
适用场景 水花、飞溅、小规模流体 海洋、河流、大规模水体

3. SPH 算法原理

3.1 核函数与场量插值

SPH 的核心思想是使用核函数将离散粒子属性平滑为连续场。任意场量 A 在位置 r 处的值可以通过邻域粒子的加权平均得到：

📐 SPH 插值公式
A(r) = Σ_j m_j/ρ_j · A_j · W(|r - r_j|, h)

其中 W 是光滑核函数，h 是光滑长度。常用 Cubic Spline 核函数保证二阶精度。

3.2 密度与压力计算

粒子密度通过核函数累加邻域粒子质量得到，然后使用状态方程计算压力： 

SPH 密度与压力计算：

密度计算：
ρ_i = Σ_j m_j · W(|x_i - x_j|, h)

压力计算（弱压缩状态方程）：
p_i = k · (ρ_i - ρ_0)

其中：
- k: 体积模量（刚度系数）
- ρ_0: 静止密度
- 常用 Tait 方程：p = B · [(ρ/ρ_0)^γ - 1]

3.3 压力梯度力

压力梯度力是保持流体不可压缩性的关键。SPH 中使用对称化的压力梯度公式：

📐 压力梯度力
f_i^pressure = -Σ_j m_j · (p_i + p_j)/(2ρ_j) · ∇W(|x_i - x_j|, h)

3.4 HLSL 实现

// Niagara SPH 流体模拟 Compute Shader

struct Particle {
    float3 position;
    float3 velocity;
    float density;
    float pressure;
    float mass;
};

RWStructuredBuffer Particles;

// Poly6 核函数
float Poly6(float r, float h) {
    float h2 = h * h;
    float h9 = h2 * h2 * h2 * h2 * h;
    float r2 = r * r;
    if (r > h) return 0;
    return 315.0 / (64.0 * PI * h9) * pow(h2 - r2, 3);
}

// Spiky 核函数梯度（用于压力）
float3 SpikyGradient(float3 r_vec, float h) {
    float r = length(r_vec);
    if (r > h || r < 0.001) return 0;
    float h6 = h * h * h * h * h * h;
    float coeff = -45.0 / (PI * h6) * pow(h - r, 2);
    return coeff * normalize(r_vec);
}

[numthreads(256, 1, 1)]
void CS_DensityPressure(uint3 id : SV_DispatchThreadID) {
    uint i = id.x;
    if (i >= NumParticles) return;
    
    Particle pi = Particles[i];
    float density = 0;
    
    // 邻域搜索（使用空间哈希优化）
    for (uint j = 0; j < NumParticles; j++) {
        if (i == j) continue;
        Particle pj = Particles[j];
        float r = distance(pi.position, pj.position);
        density += pj.mass * Poly6(r, SmoothingLength);
    }
    
    pi.density = density;
    // 状态方程：p = k * (ρ - ρ0)
    pi.pressure = Stiffness * max(0, density - RestDensity);
    
    Particles[i] = pi;
}

[numthreads(256, 1, 1)]
void CS_ForceIntegration(uint3 id : SV_DispatchThreadID) {
    uint i = id.x;
    if (i >= NumParticles) return;
    
    Particle pi = Particles[i];
    float3 force = float3(0, -Gravity, 0); // 重力
    
    // 压力梯度力
    for (uint j = 0; j < NumParticles; j++) {
        if (i == j) continue;
        Particle pj = Particles[j];
        float3 r_vec = pi.position - pj.position;
        
        float3 pressureGrad = SpikyGradient(r_vec, SmoothingLength);
        force -= pj.mass * (pi.pressure + pj.pressure) / (2 * pj.density) * pressureGrad;
    }
    
    // 积分：半隐式欧拉
    pi.velocity += force / pi.density * DeltaTime;
    pi.position += pi.velocity * DeltaTime;
    
    Particles[i] = pi;
}

4. 空间哈希加速

4.1 邻域搜索优化

直接遍历所有粒子的 O(N²) 邻域搜索在 GPU 上不可行。Niagara 使用空间哈希（Spatial Hash） 将空间划分为网格，只搜索相邻网格中的粒子，将复杂度降低到 O(N)。 

空间哈希算法：

1. 网格离散化：
   cell_coord = floor(position / cell_size)
   hash = hash(cell_coord) % hash_table_size

2. 构建哈希表：
   - 每个粒子计算其哈希值
   - 使用并行排序（Radix Sort）按哈希值排序粒子
   - 构建起始索引表：记录每个哈希值对应的粒子范围

3. 邻域查询：
   - 对于粒子i，计算其所在网格
   - 查询该网格及26个相邻网格
   - 使用起始索引表快速定位候选粒子

6.2 性能优化建议

  • • 粒子数量控制： 实时场景建议 10K-50K 粒子，高质量渲染可支持 100K+
  • • 时间步长： 满足 CFL 条件，通常 1/60s 或更小
  • • 邻域优化： 使用空间哈希，限制最大邻居数（如 50-100）
  • • LOD系统： 远距离使用粒子Billboard，近距离使用Mesh渲染
