1 概述与核心概念

什么是独立游戏技术美术？

独立游戏技术美术（Indie Technical Artist）是指在资源受限环境下，需要同时掌握渲染编程、工具开发和视觉表现的复合型人才。与3A工作室不同，独立开发者往往需要在性能预算紧张、开发周期短的约束下实现高质量的视觉效果。

🎯 核心挑战

独立游戏水面渲染面临三大核心挑战：

  • 1. 物理真实性与性能的权衡：完整的流体模拟计算成本过高，必须采用近似算法
  • 2. 跨平台一致性：从高端PC到移动设备的性能差异巨大
  • 3. 工具链整合：需要将Blender、Houdini、UE等多个软件的工作流无缝衔接

资源优化策略矩阵

在独立游戏中，资源优化不是简单的"降低质量"，而是通过算法优化和智能LOD系统实现"聪明的质量分配"。以下表格展示了不同优化层级的策略：

优化层级 策略 GPU成本 适用场景
L0: 全精度 实时FFT + 细分曲面 ~5ms 特写镜头、过场动画
L1: 高频 预计算FFT纹理 + 位移贴图 ~2ms 玩家近距离观察

💡 质量-性能权衡公式
$$Q_{total} = \sum_{i=0}^{n} w_i \cdot Q_i(L_i) \quad \text{s.t.} \quad T_{frame} < 16.67\text{ms}$$

其中 $Q_i$ 为第 $i$ 个LOD级别的视觉质量，$w_i$ 为权重系数，$T_{frame}$ 为帧时间预算。独立游戏的核心技能就是在这个约束条件下最大化 $Q_{total}$。

4 FFT海洋模拟原理

快速傅里叶变换（FFT）是实时海洋模拟的核心算法。它允许我们将复杂的波叠加问题从时域转换到频域，利用频谱的数学特性高效计算大规模波浪效果。

4.1 理论基础：线性波理论

🌊 核心洞察

海洋表面可以建模为无数个不同频率、振幅、方向的正弦波的叠加。在频域中，这些波是独立处理的，这使得FFT成为可能。线性波理论假设波幅相对于波长很小，允许使用叠加原理。

海面高度场 $\eta(\mathbf{x}, t)$ 可以表示为频域函数 $h(\mathbf{k}, t)$ 的逆傅里叶变换：

$$\eta(\mathbf{x}, t) = \sum_{\mathbf{k}} \tilde{h}(\mathbf{k}, t) \cdot e^{i\mathbf{k}\cdot\mathbf{x}}$$

其中 $\tilde{h}(\mathbf{k}, t)$ 是频域波高，$e^{i\mathbf{k}\cdot\mathbf{x}}$ 是空间振荡因子。

4.2 Phillips频谱模型

Phillips频谱是描述海面能量分布的经典模型，由Willard Phillips于1977年提出，后经Tessendorf等人改进用于计算机图形学。它基于物理观测数据，考虑了风速、风向和重力的影响。

📐 Phillips频谱公式
$$P(\mathbf{k}) = A \cdot \frac{e^{-1/(kL)^2}}{k^4} \cdot |\mathbf{k} \cdot \mathbf{D}_{wind}|^2$$
分量解析：
  • • $A$: 全局振幅控制
  • • $e^{-1/(kL)^2}$: 抑制小波长噪声
  • • $k^{-4}$: 能量随波数衰减
  • • $|\mathbf{k} \cdot \mathbf{D}|^2$: 方向性因子
物理意义：
  • • 长波（小$k$）携带更多能量
  • • 波长小于$L$的波被指数抑制
  • • 波浪倾向于沿风向传播

4.3 初始频谱生成

在时间 $t=0$ 时刻，频谱 $h_0(\mathbf{k})$ 通过Phillips频谱和随机数生成：

$$h_0(\mathbf{k}) = \frac{1}{\sqrt{2}}\left(\xi_r + i\xi_i\right) \cdot \sqrt{P(\mathbf{k})}$$

$\xi_r, \xi_i$ 是独立的高斯随机数，引入随机性以模拟自然海浪的不可预测性。$\sqrt{P(\mathbf{k})}$ 确保能量分布符合Phillips模型。

4.4 时间演化与色散关系

海浪随时间演化遵循色散关系（Dispersion Relation），描述波数与角频率的关系：

$$\omega(\mathbf{k}) = \sqrt{g \cdot |\mathbf{k}|}$$

$g \approx 9.81 m/s^2$ 为重力加速度。这个关系意味着长波传播更快，是海浪物理的核心特征。

时刻 $t$ 的频谱通过欧拉公式演化：

$$\tilde{h}(\mathbf{k}, t) = h_0(\mathbf{k}) \cdot e^{i\omega t} + h_0^*(-\mathbf{k}) \cdot e^{-i\omega t}$$

第二项是共轭对称项，确保IFFT结果为正实数。$h_0^*$ 表示复共轭。

4.5 位移与法线计算

为提高视觉真实感，除了垂直位移 $\eta$ 外，还需计算水平位移（choppiness）和法线。

水平位移场（Choppiness）：
$$D_x(\mathbf{x}, t) = \sum_{\mathbf{k}} -i\frac{k_x}{|k|} \cdot \tilde{h}(\mathbf{k}, t) \cdot e^{i\mathbf{k}\cdot\mathbf{x}}$$
$$D_z(\mathbf{x}, t) = \sum_{\mathbf{k}} -i\frac{k_y}{|k|} \cdot \tilde{h}(\mathbf{k}, t) \cdot e^{i\mathbf{k}\cdot\mathbf{x}}$$
法线计算：

法线可通过位移场的梯度直接计算，避免额外的叉乘运算：

$$\mathbf{n} = \text{normalize}\left(-\frac{\partial \eta}{\partial x}, 1, -\frac{\partial \eta}{\partial z}\right)$$
⚡ 性能优化提示

在GPU实现中，水平位移和法线计算可以复用IFFT的中间结果。通过输出多通道纹理（RGBA32F），单次FFT pass即可同时输出高度、水平位移X、水平位移Z和法线信息，显著减少显存带宽。

5 程序化水波纹理（Houdini WPO工作流）

虽然FFT提供了高质量的开阔海洋模拟，但对于中小型水体（池塘、湖泊、河流）和性能受限平台，程序化纹理动画是更实用的选择。Houdini作为程序化内容生成（PCG）的行业标准工具，可以高效生成适用于World Position Offset（WPO）的循环动画纹理。

5.1 Gerstner波理论基础

🔷 Gerstner vs 正弦波

相比简单的正弦波顶点动画，Gerstner波（也称为Trochoidal波）能生成更尖锐的波峰和更平坦的波谷，视觉上更接近真实海浪。其核心思想是：水质点沿圆形轨道运动，而非仅垂直运动。

📐 Gerstner波公式

对于第 $i$ 个波，顶点位置 $\mathbf{P} = (x, z)$ 的位移为：

$$\mathbf{P}' = \mathbf{P} - \mathbf{D}_i \cdot Q_i \cdot A_i \cdot \sin(\mathbf{D}_i \cdot \mathbf{P} \cdot w_i + t \cdot \phi_i)$$
$$y = A_i \cdot \cos(\mathbf{D}_i \cdot \mathbf{P} \cdot w_i + t \cdot \phi_i)$$
$\mathbf{D}_i$: 波传播方向
$Q_i$: 陡峭度控制 ($0 \leq Q \leq 1$)
$A_i$: 波幅
$w_i$: 频率 ($2\pi/\lambda$)
$\phi_i$: 相位速度
$t$: 时间

5.2 Houdini WPO纹理生成

Houdini的VEX（Vector Expression）语言提供了强大的程序化控制能力，可以烘焙出无缝循环的水面动画纹理。

Houdini VEX - WPO纹理生成 
// Houdini VEX: 生成WPO动画纹理
// 运行于Grid节点的Point Wrangle

// 参数定义
float Time = @Time;           // 当前时间
float LoopDuration = 4.0;     // 循环周期(秒)
int   NumWaves = 8;           // 叠加波数量
float AmpBase = 0.5;          // 基础振幅
float FreqBase = 2.0;         // 基础频率

// 计算循环时间
float loopTime = Time % LoopDuration;

// 初始化位移
vector displacement = {0, 0, 0};
vector normal = {0, 1, 0};

for(int i = 0; i < NumWaves; i++) {
    // 使用黄金角分布方向，避免波峰对齐
    float angle = i * 2.39996;  // 黄金角
    vector dir = set(cos(angle), 0, sin(angle));
    
    // 波长随索引增加而减小
    float wavelength = 2.0 * PI / (FreqBase * (1.0 + i * 0.5));
    float freq = 2.0 * PI / wavelength;
    
    // 振幅递减
    float amp = AmpBase / (1.0 + i * 0.3);
    
    // 相位速度(色散关系近似)
    float phaseSpeed = sqrt(9.81 / freq);
    float phase = freq * (dot(dir, @P) - phaseSpeed * loopTime);
    
    // Gerstner位移
    float Q = 0.8;  // 陡峭度
    float WA = freq * amp;
    
    // 水平位移
    displacement.x -= dir.x * Q * amp * sin(phase);
    displacement.z -= dir.z * Q * amp * sin(phase);
    
    // 垂直位移
    displacement.y += amp * cos(phase);
    
    // 累加法线(用于法线贴图烘焙)
    float steep = Q * WA;
    normal.x -= dir.x * steep * cos(phase);
    normal.z -= dir.z * steep * cos(phase);
}

// 输出到属性
@P += displacement;
@N = normalize(normal);

// 导出到纹理的RGB通道
@Cd = set(displacement.x * 0.5 + 0.5, 
           displacement.y * 0.5 + 0.5,
           displacement.z * 0.5 + 0.5);

5.3 无缝循环纹理烘焙

为实现无缝循环动画，需要满足以下条件：

$$\eta(\mathbf{x}, t + T_{loop}) = \eta(\mathbf{x}, t) \quad \forall \mathbf{x}, t$$

关键约束：所有波在循环周期 $T_{loop}$ 内必须完成整数个周期。这要求波的相位速度 $c_i$ 满足 $c_i \cdot T_{loop} / \lambda_i \in \mathbb{Z}$。

Houdini - 无缝循环约束计算 
// 计算满足循环约束的相位速度
// 关键：波在LoopDuration内传播整数倍波长

float LoopDuration = chf("loop_duration");  // UI参数

for(int i = 0; i < NumWaves; i++) {
    float wavelength = BaseWavelength / (i + 1);
    
    // 基础相位速度(深水波近似)
    float c_base = sqrt(9.81 * wavelength / (2 * PI));
    
    // 调整速度以满足循环约束
    // 传播距离 = c * LoopDuration
    // 需要: c * LoopDuration = N * wavelength (N为整数)
    int N = max(1, int(c_base * LoopDuration / wavelength));
    float c_adjusted = N * wavelength / LoopDuration;
    
    // 使用调整后的速度计算相位
    float phaseSpeed = c_adjusted;
}

6 实时浮力系统

浮力系统是水面交互的核心。在独立游戏中，需要在物理真实性与计算成本之间找到平衡点，通常采用简化的阿基米德原理近似。

6.1 物理基础：阿基米德原理

📐 浮力公式
$$\mathbf{F}_{buoyancy} = -\rho_{water} \cdot g \cdot V_{submerged} \cdot \mathbf{\hat{y}}$$

$\rho_{water}$: 水密度 (通常 $1000 kg/m^3$)
$V_{submerged}$: 物体浸没体积
$\mathbf{\hat{y}}$: 垂直向上单位向量

6.3 蓝图实现

UE Blueprint - 浮力组件核心逻辑 
# Unreal Engine Blueprint (Python风格伪代码)
# Buoyancy Component - Tick Event

def ApplyBuoyancy():
    # 参数配置
    WaterDensity = 1000.0        # kg/m³
    Gravity = 981.0              # cm/s² (UE单位)
    Damping = 0.3                # 阻尼系数
    
    # 获取物理组件
    Primitive = GetOwner().RootComponent
    if not Primitive.IsSimulatingPhysics():
        return
    
    TotalBuoyancyForce = Vector(0, 0, 0)
    TotalTorque = Vector(0, 0, 0)
    
    # 遍历所有采样点
    for SamplePoint in BuoyancySamplePoints:
        WorldPos = SamplePoint.GetWorldLocation()
        
        # 查询水面高度 (Blueprint Implementable Event)
        WaterHeight = GetWaterHeightAtLocation(WorldPos.X, WorldPos.Y)
        
        # 计算浸没深度
        Depth = WaterHeight - WorldPos.Z
        
        if Depth > 0:  # 在水下
            # 计算该点的浮力
            # 假设每个采样点代表固定体积 V = BuoyancyVolume / NumPoints
            SubmergedVolume = (Depth / SamplePointMaxDepth) * PointRepresentedVolume
            SubmergedVolume = Clamp(SubmergedVolume, 0, PointRepresentedVolume)
            
            # 阿基米德浮力
            BuoyancyMagnitude = WaterDensity * Gravity * SubmergedVolume
            BuoyancyForce = Vector(0, 0, BuoyancyMagnitude)
            
            # 添加阻尼力 (与速度反向)
            Velocity = Primitive.GetPhysicsLinearVelocityAtPoint(WorldPos)
            DampingForce = -Velocity * Damping * SubmergedVolume
            
            TotalForce = BuoyancyForce + DampingForce
            
            # 施加力
            Primitive.AddForceAtLocation(TotalForce, WorldPos)
            
            # 累加用于调试可视化
            TotalBuoyancyForce += TotalForce
    
    # 可选：添加额外的阻力
    Velocity = Primitive.GetPhysicsLinearVelocity()
    DragForce = -Velocity * GetDragCoefficient() * Velocity.Size()
    Primitive.AddForce(DragForce)

6.4 水面高度查询接口

浮力系统需要高效查询任意位置的水面高度。根据LOD级别，可采用不同策略：

class MYGAME_API UWaterSubsystem : public UWorldSubsystem

// 实现示例
float UWaterSubsystem::GetWaterHeightAtLocation(float X, float Y, EWaterLOD LOD)
{
    if (LOD == EWaterLOD::Auto)
    {
        // 根据距离摄像机远近自动选择LOD
        FVector CameraPos = GetWorld()->GetFirstPlayerController()->GetCameraLocation();
        float Dist = FVector2D::Distance(FVector2D(X, Y), FVector2D(CameraPos.X, CameraPos.Y));
        
        if (Dist < 1000.0f)       // 10米内
            LOD = EWaterLOD::High;
        else if (Dist < 5000.0f)  // 50米内
            LOD = EWaterLOD::Medium;
        else
            LOD = EWaterLOD::Low;
    }
    
    switch (LOD)
    {
        case EWaterLOD::High:   return QueryDisplacementMap(X, Y);
        case EWaterLOD::Medium: return QueryGerstner(X, Y);
        case EWaterLOD::Low:    return QueryFlat(X, Y);
        default: return 0.0f;
    }
}
⚠️ GPU回读的性能陷阱

从GPU读取FFT结果（RenderTarget）到CPU会触发管线停顿，严重影响性能。解决方案：

1. 异步回读: 使用Staging Texture延迟读取，允许1-2帧的延迟
2. 预测插值: 基于上一帧数据外推当前帧高度
3. 纯GPU浮力: 对非交互式物体使用Compute Shader计算浮力

// Niagara Module Script (HLSL-like syntax)
// 在Particle Update阶段执行

// 输入参数
float3 ParticlePosition;    // 当前粒子位置
float WaterLevel;          // 静态水面高度(基础)
Texture2D DisplacementMap; // FFT位移贴图
float4 DisplacementMapUV;  // 世界到UV变换

// 计算实际水面高度(含波浪)
float2 UV = ParticlePosition.xy * DisplacementMapUV.xy + DisplacementMapUV.zw;
float3 Displacement = Texture2DSample(DisplacementMap, UV).rgb;
float ActualWaterHeight = WaterLevel + Displacement.z;  // z分量是垂直位移

// 碰撞检测
bool bIsUnderWater = ParticlePosition.z < ActualWaterHeight;
bool bWasAboveWater = ParticlePosition.z - Velocity.z * DeltaTime >= ActualWaterHeight;

if (bIsUnderWater && bWasAboveWater)
{
    // 触发水面进入事件
    Interp.SpawnCount = 1;  // 生成子粒子(水花)
    
    // 计算碰撞法线
    float3 Normal;
    Normal.x = -Displacement.x;  // 水平位移导数
    Normal.z = -Displacement.y;  // 注意:纹理中y是水平位移z
    Normal.y = 1.0;
    Normal = normalize(Normal);
    
    // 反射速度
    float3 ReflectedVelocity = reflect(Velocity, Normal);
    Velocity = ReflectedVelocity * Damping;
    
    // 标记碰撞点
    CollisionPoint = ParticlePosition;
    CollisionNormal = Normal;
}
else if (bIsUnderWater)
{
    // 水下运动：添加浮力和阻力
    float3 Buoyancy = float3(0, 0, 1) * BuoyancyStrength;
    float3 Drag = -Velocity * UnderwaterDrag;
    Velocity += (Buoyancy + Drag) * DeltaTime;
}

C++ - 水面剔除优化 
// WaterMeshComponent.cpp

void UWaterMeshComponent::UpdateCullStatus()
{
    // 视锥剔除
    if (!Scene->GetViewFrustum().Intersects(BoundingBox))
    {
        bVisible = false;
        return;
    }
    
    // 距离剔除
    float Distance = (Bounds.Origin - ViewLocation).Size();
    if (Distance > MaxRenderDistance)
    {
        bVisible = false;
        return;
    }
    
    // 地平线以下剔除
    // 利用地球曲率近似快速排除远距离水面
    float HorizonDistance = 3.57 * sqrt(ViewLocation.Z / 100.0f);  // km
    if (Distance > HorizonDistance * 100000.0f && ViewLocation.Z < 1000.0f)
    {
        bVisible = false;
        return;
    }
    
    // 遮挡查询 (使用前一帧深度图)
    if (bUseOcclusionCulling)
    {
        FBox2D ScreenBox = ProjectBoxToScreen(BoundingBox);
        float OcclusionRatio = QueryOcclusion(ScreenBox);
        if (OcclusionRatio < 0.05f)  // 95%被遮挡
        {
            bVisible = false;
            return;
        }
    }
    
    bVisible = true;
}

// FFTOceanComponent.cpp - 核心实现节选

void UFFTOceanComponent::InitializeRenderTargets()
{
    // 创建渲染目标
    auto CreateRT = [&](FName Name, bool bHDR = true) -> UTextureRenderTarget2D*
    {
        UTextureRenderTarget2D* RT = NewObject(this, Name);
        RT->InitCustomFormat(FFTResolution, FFTResolution, PF_FloatRGBA, bHDR);
        RT->AddressX = TA_Wrap;
        RT->AddressY = TA_Wrap;
        RT->Filter = TF_Bilinear;
        RT->UpdateResource();
        return RT;
    };
    
    InitialSpectrumTarget = CreateRT(FName("InitialSpectrum"));
    TimeSpectrumTarget = CreateRT(FName("TimeSpectrum"));
    DisplacementTarget = CreateRT(FName("Displacement"));
    NormalTarget = CreateRT(FName("Normal"));
    
    // 初始化高斯随机数 (用于Phillips谱)
    GaussianRandoms.Reserve(FFTResolution * FFTResolution);
    FRandomStream RandStream(12345);  // 固定种子保证可重复性
    
    for (int32 i = 0; i < FFTResolution * FFTResolution; i++)
    {
        // Box-Muller变换生成高斯分布
        float U1 = RandStream.GetFraction();
        float U2 = RandStream.GetFraction();
        float R = FMath::Sqrt(-2.0f * FMath::Loge(U1));
        float Theta = 2.0f * PI * U2;
        
        GaussianRandoms.Add(FVector2D(
            R * FMath::Cos(Theta),
            R * FMath::Sin(Theta)
        ));
    }
    
    bInitialized = true;
}

float UFFTOceanComponent::PhillipsSpectrum(FVector2D K) const
{
    float KLength = K.Size();
    if (KLength < 0.0001f) return 0.0f;
    
    // Phillips谱参数
    float L = WindSpeed * WindSpeed / 981.0f;  // 特征波长 (g = 981 cm/s²)
    FVector2D KNorm = K / KLength;
    FVector2D WNorm = WindDirection.GetSafeNormal();
    
    // 方向因子
    float DotKW = FVector2D::DotProduct(KNorm, WNorm);
    float DirectionalFactor = FMath::Pow(DotKW, 2);
    
    // 抑制与风向垂直的波
    if (DotKW < 0.0f) DirectionalFactor *= 0.07f;
    
    // Phillips谱公式
    float K2 = KLength * KLength;
    float K4 = K2 * K2;
    float L2 = L * L;
    
    float P = Amplitude * FMath::Exp(-1.0f / (K2 * L2)) / K4;
    P *= DirectionalFactor;
    
    // 抑制小波长噪声
    float KDotK = KLength * 0.0005f;  // 调整平滑因子
    P *= FMath::Exp(-K2 * KDotK * KDotK);
    
    return P;
}
