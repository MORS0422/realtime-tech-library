MegaLights：千光源实时渲染的随机采样与重要性采样架构

Unreal Engine 5.5 引入的 MegaLights 是实时渲染领域光照计算的范式突破。传统延迟渲染（Deferred Shading） 面临的光源数量限制（通常<100个动态光源）在 MegaLights 的随机采样架构下被彻底打破。 通过 Blue Noise 抖动采样、时域累积（Temporal Accumulation）和重要性采样（Importance Sampling）的协同， MegaLights 实现了数千盏动态光源的实时渲染，为夜景、室内场景和特效密集场景提供了全新的可能性。

🔬 核心技术突破

MegaLights 的核心创新是随机采样策略。对于每个像素，不再计算所有光源的贡献， 而是根据光源重要性进行概率采样。使用 Blue Noise 抖动模式减少视觉伪影，通过时域累积整合多帧结果。 性能测试显示，100盏光源时 MegaLights 比传统方案快2.7倍，500盏时快7倍，2000盏时传统方案已不可用而 MegaLights 仅需12ms。 这对于霓虹灯闪烁的赛博朋克场景、烛光摇曳的中世纪地牢等复杂光照环境具有革命性意义。

1. 概述与核心概念

1.1 实时光照的技术挑战

实时渲染中的光照计算面临一个根本性的困境：光照质量与计算成本之间的矛盾。 渲染方程要求对所有入射光方向进行积分，但在有限的时间内只能采样有限数量的光源。

⚠️ 传统延迟渲染的瓶颈
  • • 光源数量限制：传统Tiled/Clustered Deferred通常限制在100-200个动态光源
  • • 计算复杂度O(N)：每个像素需要遍历所有影响该像素的光源
  • • 带宽瓶颈：光源数据从CPU到GPU的传输成为瓶颈
  • • 阴影开销：每盏光源都需要独立的阴影图计算

1.2 蒙特卡洛积分与随机采样

MegaLights 采用蒙特卡洛积分（Monte Carlo Integration）来估算光照积分。 核心思想是：通过随机采样少量光源，利用统计学原理估计整体光照效果。

📐 光照积分与蒙特卡洛估计

精确的光照积分需要对场景中的所有光源求和：

L(x, ω_o) = Σ_{i=1}^{N} f_r(ω_i, ω_o) · L_i · G(x, x_i) · V(x, x_i)

其中 N 是光源总数。当 N 很大时，计算成本过高。蒙特卡洛估计通过采样 k 个光源近似：

L(x, ω_o) ≈ (1/k) Σ_{j=1}^{k} [f_r(ω_j, ω_o) · L_j · G(x, x_j) · V(x, x_j)] / p(j)

其中 p(j) 是采样光源 j 的概率密度函数（PDF）。

1.3 重要性采样（Importance Sampling）

朴素随机采样会导致高方差（噪声），因为每个光源被采样的概率相等。 重要性采样根据光源对最终像素的贡献程度调整采样概率， 让更重要的光源被更频繁地采样。

📐 重要性采样PDF

光源 i 的采样概率与其重要性权重成正比：

p(i) = w_i / Σ_{j=1}^{N} w_j

重要性权重 w_i 通常考虑以下因素：

w_i = Intensity_i · Attenuation(distance_i) · cos(θ_i) · f_r(ω_i, ω_o)

3. 核心理论：随机光照采样系统

3.1 Stochastic Tiled Deferred Shading

MegaLights 在 Tiled Deferred 的基础上引入随机采样。 屏幕被划分为固定大小的Tile（通常为16x16像素），每个Tile独立进行光源采样。 

Stochastic Tiled Shading流程：

阶段1：光源列表构建（Compute Shader）
├─ 对每个Tile：
│   ├─ 计算Tile的视锥
│   ├─ 测试所有光源与Tile的相交
│   └─ 生成影响该Tile的光源列表

阶段2：重要性权重计算
├─ 对每个像素：
│   ├─ 计算到各光源的距离
│   ├─ 计算光源强度衰减
│   ├─ 计算BRDF响应
│   └─ 综合计算重要性权重 w_i

阶段3：随机采样
├─ 根据权重分布生成采样概率 p(i) = w_i / Σw
├─ 使用Blue Noise生成随机数
├─ 按概率采样k个光源
└─ 计算蒙特卡洛估计

阶段4：时域累积
├─ 当前帧结果与历史帧混合
├─ 处理遮挡变化和光源移动
└─ 输出最终光照

3.2 Blue Noise抖动采样

使用Blue Noise而非White Noise进行抖动采样，可以显著减少视觉伪影。 Blue Noise的特点是在低频区域能量分布均匀，高频区域有随机性，这与人眼视觉特性匹配。

特性 White Noise Blue Noise
频谱特性 全频段均匀分布 低频少，高频多
视觉表现 颗粒感强，有明显噪点 更自然，类似胶片颗粒
时域累积 收敛较慢 收敛更快，残留噪声更少
相邻像素 采样独立，可能出现色差 采样相关，保持平滑

3.3 时域累积（Temporal Accumulation）

每帧只采样少量光源会产生明显的噪声。时域累积通过混合多帧的采样结果， 在时间上积累足够的样本，达到降噪目的。

📐 时域累积公式
L_acc(t) = α_blend · L_acc(t-1) + (1 - α_blend) · L_curr(t)

其中：

  • • L_acc(t-1)：历史累积结果
  • • L_curr(t)：当前帧采样结果
  • • α_blend：混合因子（通常0.9-0.95）

时域抗锯齿与重投影

当相机或物体移动时，需要正确地将历史样本重投影到当前帧。 这涉及运动向量的计算和历史样本有效性的验证。 

重投影与有效性验证：

1. 运动向量计算
   ├─ 根据相机运动和物体动画计算每像素的运动向量
   └─ velocity = (curr_pos - prev_pos) / delta_time

2. 历史样本查找
   ├─ history_uv = curr_uv - velocity * delta_time
   └─ 采样历史缓冲

3. 有效性验证
   ├─ 深度差异检查：|depth_curr - depth_hist| < threshold
   ├─ 法线差异检查：dot(normal_curr, normal_hist) > threshold
   ├─ 光源变化检测：光源位置/强度是否发生变化
   └─ 遮挡变化：当前可见但历史不可见（或相反）

4. 无效样本处理
   └─ 当历史样本无效时，降低α_blend（更快收敛到新状态）

3.4 光源重要性采样算法

重要性采样的核心是构建光源的累积分布函数（CDF）， 然后通过均匀随机数在CDF上二分查找来确定采样光源。

📐 重要性权重计算

光源i的完整重要性权重：

w_i = I_i · f_att(d_i) · max(0, n·l_i) · BRDF(l_i, v)

其中各项含义：

  • I_i: 光源i的基础强度
  • f_att(d_i) = 1/(d_i²): 距离平方衰减
  • max(0, n·l_i): 朗伯余弦定律
  • BRDF(l_i, v): 双向反射分布函数
CDF构建与采样：

1. 权重归一化
   W_total = Σ_{i=1}^{N} w_i
   p(i) = w_i / W_total

2. 累积分布函数
   CDF[0] = p(0)
   CDF[i] = CDF[i-1] + p(i)
   
   性质：CDF[N-1] = 1.0

3. 采样过程
   ├─ 生成均匀随机数 ξ ∈ [0, 1)
   ├─ 在CDF数组中二分查找满足 CDF[i-1] ≤ ξ < CDF[i] 的i
   └─ 返回光源索引i作为采样结果

4. 多重采样
   ├─ 对每个像素采样k个光源
   ├─ 使用k个不同的随机数（通过Blue Noise偏移）
   └─ 累加各样本的贡献，除以k得到估计值

4. 算法实现

4.1 光源列表构建Compute Shader

// MegaLights 光源列表构建

struct LightData {
    float3 position;
    float intensity;
    float3 color;
    float radius;
    uint type;  // 0=point, 1=spot, 2=area
};

StructuredBuffer Lights;
RWStructuredBuffer TileLightList;  // 每个Tile的光源索引列表
RWStructuredBuffer TileLightCount; // 每个Tile的光源数量

// Tile参数
#define TILE_SIZE 16
#define MAX_LIGHTS_PER_TILE 256

[numthreads(TILE_SIZE, TILE_SIZE, 1)]
void BuildLightList(uint3 id : SV_DispatchThreadID, uint3 gid : SV_GroupID) {
    uint tileIndex = gid.y * TileCountX + gid.x;
    uint2 tilePixel = gid.xy * TILE_SIZE;
    
    // 计算Tile的视锥
    Frustum tileFrustum = ComputeTileFrustum(tilePixel, TILE_SIZE);
    
    // 遍历所有光源，测试与Tile的相交
    uint lightCount = 0;
    for (uint i = 0; i < NumLights; i++) {
        LightData light = Lights[i];
        
        if (LightIntersectsFrustum(light, tileFrustum)) {
            uint slot;
            InterlockedAdd(TileLightCount[tileIndex], 1, slot);
            if (slot < MAX_LIGHTS_PER_TILE) {
                TileLightList[tileIndex * MAX_LIGHTS_PER_TILE + slot] = i;
            }
        }
    }
}

4.2 重要性采样与光照计算

// MegaLights 重要性采样光照计算

Texture2D GBufferAlbedo;
Texture2D GBufferNormal;
Texture2D GBufferRoughness;
Texture2D BlueNoiseTexture;

float3 CalculateMegaLights(
    float3 worldPos, 
    float3 normal, 
    float3 albedo, 
    float roughness,
    uint2 pixelCoord,
    uint tileIndex
) {
    // 获取影响当前Tile的光源列表
    uint lightCount = TileLightCount[tileIndex];
    if (lightCount == 0) return float3(0, 0, 0);
    
    // 计算每个光源的重要性权重
    float weights[MAX_LIGHTS_PER_TILE];
    float totalWeight = 0.0;
    
    for (uint i = 0; i < lightCount; i++) {
        uint lightIdx = TileLightList[tileIndex * MAX_LIGHTS_PER_TILE + i];
        LightData light = Lights[lightIdx];
        
        // 计算光源贡献权重
        float3 toLight = light.position - worldPos;
        float distance = length(toLight);
        float3 lightDir = toLight / distance;
        
        // 距离衰减
        float attenuation = 1.0 / (distance * distance);
        attenuation *= saturate(1.0 - distance / light.radius);
        
        // 角度衰减（朗伯）
        float NdotL = saturate(dot(normal, lightDir));
        
        // BRDF响应（简化GGX）
        float brdf = GGX_BRDF(normal, viewDir, lightDir, roughness);
        
        // 综合权重
        weights[i] = light.intensity * attenuation * NdotL * brdf;
        totalWeight += weights[i];
    }
    
    // 归一化权重
    for (uint j = 0; j < lightCount; j++) {
        weights[j] /= totalWeight;
    }
    
    // 构建CDF
    float cdf[MAX_LIGHTS_PER_TILE];
    cdf[0] = weights[0];
    for (uint k = 1; k < lightCount; k++) {
        cdf[k] = cdf[k-1] + weights[k];
    }
    
    // Blue Noise抖动采样
    float2 noiseUV = (float2(pixelCoord) + FrameIndex * float2(19, 47)) % 64.0;
    float noiseValue = BlueNoiseTexture.SampleLevel(PointSampler, noiseUV / 64.0, 0).r;
    
    // 采样k个光源
    float3 accumulatedLight = float3(0, 0, 0);
    uint numSamples = 2;  // 每帧采样2个光源
    
    for (uint s = 0; s < numSamples; s++) {
        // 使用分层采样
        float xi = frac(noiseValue + s / float(numSamples));
        
        // CDF二分查找
        uint sampledIdx = 0;
        for (uint idx = 0; idx < lightCount; idx++) {
            if (xi < cdf[idx]) {
                sampledIdx = idx;
                break;
            }
        }
        
        uint lightIdx = TileLightList[tileIndex * MAX_LIGHTS_PER_TILE + sampledIdx];
        LightData light = Lights[lightIdx];
        
        // 计算该光源的贡献
        float3 lightContrib = EvaluateLight(light, worldPos, normal, albedo, roughness);
        
        // 可见性测试（阴影）
        float visibility = TraceShadowRay(worldPos, light.position);
        
        // 蒙特卡洛权重 = 实际贡献 / 采样概率
        float pdf = weights[sampledIdx];
        accumulatedLight += lightContrib * visibility / pdf / numSamples;
    }
    
    return accumulatedLight;
}

4.3 时域累积实现

// 时域累积降噪

Texture2D CurrentFrameLighting;
Texture2D HistoryLighting;
Texture2D MotionVectors;
Texture2D CurrentDepth;
Texture2D HistoryDepth;
Texture2D CurrentNormal;
Texture2D HistoryNormal;

float3 TemporalAccumulation(uint2 pixelCoord) {
    float3 current = CurrentFrameLighting[pixelCoord].rgb;
    float depth = CurrentDepth[pixelCoord];
    float3 normal = CurrentNormal[pixelCoord];
    
    // 重投影
    float2 motion = MotionVectors[pixelCoord];
    float2 historyUV = (float2(pixelCoord) + 0.5) / ScreenSize + motion;
    
    // 边界检查
    if (any(historyUV < 0.0) || any(historyUV > 1.0)) {
        return current;  // 无有效历史样本
    }
    
    // 采样历史
    float3 history = HistoryLighting.SampleLevel(LinearSampler, historyUV, 0).rgb;
    float historyDepth = HistoryDepth.SampleLevel(PointSampler, historyUV, 0).r;
    float3 historyNormal = HistoryNormal.SampleLevel(PointSampler, historyUV, 0).rgb;
    
    // 有效性验证
    float depthDiff = abs(depth - historyDepth) / depth;
    float normalDiff = 1.0 - dot(normal, historyNormal);
    
    bool valid = depthDiff < 0.1 && normalDiff < 0.1;
    
    if (!valid) {
        // 无效历史，快速收敛
        return current;
    }
    
    // 颜色裁剪（Color Clamping）
    // 计算当前像素邻域的最小/最大值
    float3 minColor = current, maxColor = current;
    for (int x = -1; x <= 1; x++) {
        for (int y = -1; y <= 1; y++) {
            float3 neighbor = CurrentFrameLighting[pixelCoord + int2(x, y)];
            minColor = min(minColor, neighbor);
            maxColor = max(maxColor, neighbor);
        }
    }
    
    // 将历史样本裁剪到有效范围内
    history = clamp(history, minColor, maxColor);
    
    // 时域混合
    float blendAlpha = 0.9;  // 90%历史，10%当前
    return lerp(current, history, blendAlpha);
}

5. 性能优化策略

5.1 自适应采样数

不是所有像素都需要相同的采样数。根据场景的复杂度自适应调整每像素的采样数， 可以显著提高性能。

自适应采样策略

  • • 简单区域： 只有1-2个主要光源影响的区域，每帧采样1个
  • • 复杂区域： 有多个中等强度光源的区域，每帧采样2-4个
  • • 边缘区域： 光源变化剧烈的区域，增加采样数
  • • 运动区域： 物体或相机快速移动区域，降低历史权重

5.2 分层光源管理

光源分层策略：

层级1：重要光源（Top Tier）
├─ 太阳/主方向光
├─ 玩家手中的光源
├─ 距离近且强度高的光源
└─ 处理方式：每帧必算，可能使用传统阴影图

层级2：中等光源（Mid Tier）
├─ 场景中的装饰性光源
├─ 中等距离的点光源
└─ 处理方式：MegaLights随机采样

层级3：环境光源（Ambient Tier）
├─ 远距离的光源
├─ 强度低的小光源
└─ 处理方式：合并为环境项，或使用极低采样率

5.3 硬件光追阴影回退

对于最重要的光源，MegaLights 可以回退到硬件光线追踪获取更精确的阴影。 这种混合策略在性能和精度之间取得平衡。 

// 混合阴影策略

float GetShadow(float3 worldPos, uint lightIdx, float importance) {
    const float RT_FALLBACK_THRESHOLD = 0.8;
    
    if (importance > RT_FALLBACK_THRESHOLD && HardwareRTAvailable) {
        // 重要光源使用硬件光追阴影
        return TraceRayShadow(worldPos, Lights[lightIdx].position);
    } else {
        // 其他光源使用VSM或简化阴影
        return SampleVSMShadow(worldPos, lightIdx);
    }
}
