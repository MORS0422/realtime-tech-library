Lumen 全局光照系统：实时光线追踪的软件与硬件协同

Lumen 是 Unreal Engine 5 的革命性全局光照系统，它标志着实时渲染从预计算时代迈向完全动态时代。 通过软件光线追踪（基于有符号距离场）与硬件光线追踪（RT Core）的混合架构，Lumen 实现了影视级全局光照、 反射和软阴影的实时计算，彻底摆脱了光照贴图烘焙的束缚。

🔬 核心创新

Lumen 的核心突破在于多层级光线追踪架构。对于近处几何，使用高精度网格光追； 对于远处和复杂场景，使用全局距离场（Global SDF）进行快速光线步进。这种混合策略在保证质量的同时， 将光线追踪开销控制在可接受的范围内，使中高端GPU（RTX 2070及以上）能够实现60fps的实时渲染。

1. 概述与核心概念

1.1 什么是全局光照（Global Illumination）？

全局光照（GI）是指光线在场景中经过无限次反弹后到达人眼的综合效果。与仅计算直接光照（光源→表面）不同， GI 考虑了间接光照：光线从一个表面反射/折射后，照亮其他表面的过程。这是实现真实感的核心要素。

🎯 渲染方程

全局光照的数学基础是渲染方程（Rendering Equation）： 

L_o(p, ω_o) = L_e(p, ω_o) + ∫_Ω f_r(p, ω_i, ω_o) · L_i(p, ω_i) · (n·ω_i) dω_i

其中：
- L_o: 出射辐射亮度
- L_e: 自发光亮度  
- f_r: BRDF（双向反射分布函数）
- L_i: 入射亮度
- n·ω_i: 入射角余弦（几何衰减）

1.2 GI 技术演进

发展阶段 技术方案 特点
烘焙时代 (2000-2015) Lightmap、Light Probe 预计算，静态场景，无法支持动态光源
屏幕空间 (2015-2020) SSAO、SSR、SSGI 实时但仅限屏幕内信息，遮挡外信息丢失
体素/探针 (2018-2022) VXGI、DDGI、RTX GI 体素化场景或探针网格，中等质量
全动态光追 (2022-) Lumen、路径追踪 完全动态，无限反弹，实时性能

2. 符号定义表

2.1 几何与光线

符号 定义 说明
p 表面点 着色点在三维空间中的坐标
ω_i 入射方向 指向光源的单位向量
ω_o 出射/视线方向 指向观察者的单位向量
n 表面法线 单位法向量，n·ω_i = cos(θ)
h 半程向量 h = normalize(ω_i + ω_o)，用于镜面反射计算

2.2 Lumen 特定参数

符号 定义 说明
SDF(p) 有符号距离场 p到最近表面的有符号距离，负值表示内部
R_r 射线步进范围 Lumen光线追踪的最大距离（默认20米）
θ_r 射线半球 采样射线的立体角范围
N_s 每帧采样数 每个像素每帧的GI采样射线数量

3. Lumen 架构原理

3.1 多层级光线追踪

Lumen 采用分层光线追踪策略，根据距离和重要性选择不同的追踪方法： 

Lumen 光线追踪层级：

第1层：屏幕空间光线 (Screen Traces)
- 范围：0-2米
- 方法：屏幕空间光线步进（类似SSR）
- 优点：高精度，捕捉细节

第2层：网格距离场 (Mesh Distance Fields)
- 范围：2-20米
- 方法：每个网格的局部SDF
- 优点：精确的几何表示

第3层：全局距离场 (Global Distance Field)
- 范围：20米以外
- 方法：世界空间的统一SDF
- 优点：快速，适合远距离

第4层：硬件光追 (Hardware RT)
- 范围：全场景（可选）
- 方法：RT Core加速
- 优点：最高精度，支持复杂几何

3.2 有符号距离场（SDF）

有符号距离场（Signed Distance Field）是 Lumen 软件光追的核心数据结构。 SDF 定义了空间中任意点到最近表面的距离，符号表示内外关系。

📐 SDF 定义
SDF(p) = min_{q∈Surface} ||p - q|| · sign(p)

其中 sign(p) 在物体外部为+1，内部为-1。SDF 允许使用球体追踪（Sphere Tracing）进行高效光线步进。

球体追踪算法

// 球体追踪（Sphere Tracing）
float3 RaymarchSDF(float3 ro, float3 rd, float maxDist) {
    float t = 0.0;
    for (int i = 0; i < MAX_STEPS; i++) {
        float3 p = ro + rd * t;
        float d = SDF(p);  // 查询SDF
        
        if (d < EPSILON)   // 命中表面
            return p;
        
        t += d;            // 安全步进
        
        if (t > maxDist)   // 超出范围
            break;
    }
    return float3(0,0,0);  // 未命中
}

3.3 全局光照采样

Lumen 使用蒙特卡洛积分来估算渲染方程。由于实时性能限制， 每帧只能发射少量采样射线（通常4-8根），通过时域累积（TAA）来降噪。

📐 蒙特卡洛GI估计
L_indirect(p, ω_o) ≈ (1/N) Σ_{k=1}^N [f_r(ω_k, ω_o) · L_k · (n·ω_k)] / pdf(ω_k)

N为每帧采样数，pdf为采样方向的PDF。Lumen使用重要性采样，优先采样BRDF高光方向。

4. 反射与镜面光照

4.1 镜面反射追踪

Lumen 支持镜面反射（Specular Reflections），通过追踪反射射线来获取环境反射信息。 对于光滑表面，Lumen 可以追踪清晰的反射；对于粗糙表面，则进行锥形追踪（Cone Tracing）来模拟模糊反射。 

反射粗糙度处理：

光滑表面 (Roughness < 0.2):
- 单根镜面反射射线
- 清晰的反射图像

中等粗糙 (0.2 < Roughness < 0.5):
- 多根散射射线
- 半模糊的反射

粗糙表面 (Roughness > 0.5):
- 锥形追踪（Cone Tracing）
- 漫反射风格的反射

5. 性能优化

5.1 时域累积与降噪

Lumen 使用时域抗锯齿（TAA）来累积多帧的GI结果，将每帧少量采样扩展为有效的大量采样。 这需要解决遮挡变化（物体移动）和光照变化（光源动态）带来的历史样本失效问题。

Lumen 性能参数推荐

  • • 质量模式（Quality）： 每帧8根射线，20米范围，适合过场动画
  • • 性能模式（Performance）： 每帧4根射线，10米范围，适合60fps游戏
  • • 终极模式（Epic）： 每帧16根射线，50米范围，适合展示/影视

5.2 平台优化策略

平台 推荐设置 预期性能
RTX 4090 / RX 7900 XTX 硬件光追开启，Epic质量 4K@60fps
RTX 3070 / RX 6700 XT 软件光追，高质量 1440p@60fps
RTX 2060 / GTX 1080 软件光追，性能模式 1080p@30-60fps
PlayStation 5 / Xbox X 硬件光追，优化设置 4K@30fps / 1440p@60fps

6. 实战应用

6.1 Lumen 启用与配置

// 在项目设置中启用Lumen
// Edit -> Project Settings -> Engine - Rendering

// 动态全局光照方法
r.DynamicGlobalIlluminationMethod=1  // 1 = Lumen

// 反射方法
r.ReflectionMethod=1  // 1 = Lumen

// 质量设置（通过控制台命令）
// 软件光追质量
r.Lumen.ScreenProbeGather.RadianceCache.Resolution=32
r.Lumen.ScreenProbeGather.TracingRadianceCache.ProbeResolution=32

// 硬件光追设置（如可用）
r.Lumen.HardwareRayTracing=1
r.Lumen.HardwareRayTracing.PullbackBias=20

6.2 材质注意事项

  • • 自发光（Emissive）： Lumen 支持自发光材质作为光源，但需要在材质中启用"影响动态光照"
  • • 透明材质： 当前版本的 Lumen 对透明/折射材质支持有限，建议使用传统光照方案
  • • 双面材质： 确保双面材质的背面正确生成距离场，否则可能出现漏光
