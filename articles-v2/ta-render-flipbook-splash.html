1. 概述与核心概念

1.1 Flipbook技术原理

Flipbook（序列帧动画）是实时渲染中一种经典且高效的特效技术，其核心思想是将复杂的动态效果预渲染为一系列静态图像帧，通过快速顺序播放这些帧来模拟连续动画。在游戏VFX领域，Flipbook占据了极其重要的地位，特别是在粒子系统表现流体、爆炸、火焰等复杂物理现象时。

🔬 技术深度分析

Flipbook的本质是时间-空间权衡（Time-Space Trade-off）。通过预先计算并存储时间维度上的状态变化，我们在运行时换取了极致的计算效率。一张512×512、包含64帧的Flipbook纹理，在着色器中仅需一次纹理采样即可获取完整动画信息，避免了实时求解Navier-Stokes方程的巨大开销。这种技术选择体现了游戏渲染中"烘焙的艺术"——用内存带宽换取计算带宽。

1.2 方向性控制的核心矛盾

在水花特效制作中，方向性控制是一个经典的技术难题。当使用Houdini的FLIP求解器模拟具有明确方向性的水花（如角色移动时产生的尾流、爆炸冲击波、物体落入水中的定向飞溅）时，我们面临一个根本性的矛盾：

核心问题

当施加定向力场（如velocity field）引导流体时，粒子会在模拟空间中发生物理位移。但在Flipbook渲染中，相机需要保持固定视角以捕捉特定范围的效果。这导致物理模拟的真实运动与渲染帧的固定视锥产生冲突——要么粒子跑出画面，要么必须拉远相机导致特效在画面中变小。

1.4 解决方案概览

针对方向性控制问题，业界发展出多种互补的技术方案：

  • 相机拉远 + 引擎缩放：扩展渲染范围后在材质中放大补偿
  • 非正方形帧：使用矩形UV布局匹配特效的长宽比
  • RGBA通道打包：将灰度序列帧打包到单个纹理的RGBA通道
  • VAT替代方案：使用Vertex Animation Texture实现真正的3D变形

3.2 各阶段详解

阶段1：Setup（初始化）

配置FLIP求解器参数，定义模拟域Ω的大小和分辨率，设置碰撞体边界条件，初始化粒子分布。方向性控制的关键在于参考系的选取——此时需要决定是否使用跟随相机。

阶段2：Simulate（模拟）

FLIP求解器执行核心算法循环：Advection → Force Integration → Pressure Projection。此阶段施加定向力场F来引导水花形状。帧范围通常需要覆盖完整的生命周期（60-120帧@24fps）。

阶段3：Cache（缓存）

将粒子数据转换为可渲染格式。通常导出为bgeo或vdb格式，包含粒子位置x、速度u、密度ρ等属性。对于体积渲染，需要将粒子转换为密度场。

阶段4：Render（渲染）

使用Mantra或其他渲染器逐帧渲染。相机设置决定Flipbook的视觉效果。针对方向性问题，需要计算合适的camera frustum以包含完整运动轨迹。

阶段5：Post-Process（后处理）

生成附加数据通道：Normal Map用于光照计算，Motion Vectors用于运行时运动模糊，Alpha通道用于边缘平滑。最后进行UV布局和通道打包。

3.3 方向性控制的技术切入点

在完整管线的视角下，方向性控制问题有多个技术切入点：

切入点 技术方案 优点 缺点
模拟阶段 力场约束、参考系转换 物理一致性最好 实现复杂
渲染阶段 拉远相机、扩展frustum 直观易实现 texel密度降低
后处理阶段 非正方形UV、通道打包 优化内存效率 不解决根本问题
运行时 引擎端缩放、VAT替代 完全避免问题 增加渲染开销

4. Houdini流体模拟理论

4.1 FLIP算法数学基础

FLIP（Fluid-Implicit-Particle）是Houdini流体模拟的核心算法，由Zhu和Bridson在2005年提出。它结合了PIC（Particle-In-Cell）和粒子方法的优点，实现了高效且稳定的流体模拟。

🔬 算法原理深度解析

FLIP算法的核心创新在于双重表示（Dual Representation）：使用粒子携带流体质点的速度信息，同时借助欧拉网格求解压力投影。这种混合方法巧妙地规避了纯拉格朗日方法的邻居搜索开销，以及纯欧拉方法的对流数值耗散。

算法的关键步骤是速度增量传输：FLIP将网格计算出的速度变化量插值回粒子，而非像PIC那样直接传输速度绝对值。这一细微差别使得FLIP能保留更多的速度场细节，特别适用于模拟水花这类需要高湍流细节的效果。

4.1.1 Navier-Stokes方程

不可压缩流体的运动由Navier-Stokes方程描述：

动量方程（Momentum Equation）
∂u/∂t + (u · ∇)u = -∇p/ρ + ν∇²u + F/ρ
连续性方程（Continuity Equation）
∇ · u = 0

其中各项的物理意义：

  • ∂u/∂t：局部加速度（Local acceleration）
  • (u · ∇)u：对流项（Advection term），表示流体输运自身速度
  • -∇p/ρ：压力梯度力，维持不可压缩性的关键
  • ν∇²u：粘性扩散项，描述动量耗散
  • F/ρ：外力项（重力、风力、自定义力场）

4.1.2 FLIP时间积分

FLIP算法采用算子分裂（Operator Splitting）策略，将复杂方程分解为可顺序求解的步骤：

Step 1: Advection（对流）
u* = advect(un, un, h)
Step 2: Force Integration（外力积分）
u** = u* + h · F/ρ
Step 3: Pressure Projection（压力投影）
Solve: ∇²p = (ρ/h) · ∇ · u**
un+1 = u** - (h/ρ) · ∇p

4.2 速度场（Velocity Field）

速度场u(x, t)是流体模拟的核心数据结构。在FLIP中，它以两种形式存在：

4.2.1 粒子速度 vs 网格速度

表示形式 存储位置 优势 适用场景
uparticle 粒子属性@v 无数值耗散 细节保留、湍流
ugrid 体素中心 便于求解压力 投影、边界处理

4.2.2 PIC/FLIP混合方案

纯FLIP方法在高分辨率模拟中可能产生噪声。实际应用中采用PIC/FLIP混合：

ufinal = α · uFLIP + (1-α) · uPIC
其中 α ∈ [0, 1] 为混合系数

Houdini参数：FLIP Solver → Particle Motion → Flip/Pic Ratio
默认值0.95（95% FLIP），降低此值可增加稳定性但会减少细节。

4.3 方向性力场设计

为实现特定方向的水花形态，需要在模拟阶段施加精心设计的力场。力场的设计直接影响最终效果的视觉特征。

4.3.1 力场类型

Uniform Force（均匀力场）

F = c（常数向量）
适用于模拟重力、恒定风力。所有粒子受到相同方向和大小的力。

Radial Force（径向力场）

F(x) = f(|x - xcenter|) · (x - xcenter) / |x - xcenter|
爆炸、扩散效果。力的大小随距离变化，方向沿径向。

Directional Force with Falloff（衰减定向力）

F(x) = d · exp(-|x - x0|² / 2σ²)
高斯分布的力场，在中心区域最强，向边缘衰减。适用于局部冲击效果。

Vortex Force（涡旋力场）

F(x) = ω × (x - xcenter) · g(r)
产生旋转运动，ω为角速度向量，g(r)为径向衰减函数。

5. 方向控制技术

针对Flipbook水花制作中的方向性控制难题，本节详细讲解四种核心技术方案，从模拟层面的力场引导到渲染层面的相机策略。

5.1 力场引导（Force Field Guidance）

5.1.1 基础定向力实现

最直接的方法是在模拟空间施加定向力，同时通过其他约束防止粒子逃离渲染范围。 

// VEX: popwrangle节点中的定向力
vector force_dir = chv("force_direction");  // 力场方向
float force_mag = chf("force_magnitude");    // 力场强度
float decay = chf("distance_decay");         // 距离衰减

// 计算距离中心的距离
float dist = length(@P - chv("center"));
float falloff = exp(-decay * dist);

// 应用带衰减的定向力
vector force = normalize(force_dir) * force_mag * falloff;
@v += force * @TimeInc;

5.1.2 边界约束策略

为了防止粒子跑出相机视野，可以添加软边界约束（Soft Boundary Constraint）：

Fboundary = -k · (x - xlimit) · H(x - xlimit)
其中 H 为Heaviside阶跃函数，k 为弹簧常数 
// VEX: 软边界约束
vector bbox_min = chv("bbox_min");
vector bbox_max = chv("bbox_max");
float stiffness = chf("stiffness");

// 计算边界恢复力
vector restore = 0;
for(int i = 0; i < 3; i++) {
    if(@P[i] < bbox_min[i]) {
        restore[i] = stiffness * (bbox_min[i] - @P[i]);
    }
    if(@P[i] > bbox_max[i]) {
        restore[i] = -stiffness * (@P[i] - bbox_max[i]);
    }
}

@v += restore * @TimeInc;
⚠️ 技术权衡分析

边界约束虽然能防止粒子逃逸，但会引入非物理行为。当恢复力过强时，水花在边界处会产生不自然的"反弹"或堆积；过弱则约束失效。最佳实践是在模拟初期允许粒子自由运动，在粒子速度衰减后（如生命周期60%以后）逐渐增强边界约束。

5.2 Mask控制（Spatial Masking）

5.2.1 基于Volume Mask的力场控制

使用Volume Field定义力的空间分布，实现更精细的控制： 

// VEX: 读取Volume Mask并应用力场
// 假设存在名为"mask"的volume和名为"force_field"的volume

float mask = volumesample(1, "mask", @P);
vector field = volumesamplev(1, "force_field", @P);

// Mask控制力的应用强度
@v += field * mask * @TimeInc;

5.2.2 时间变化的Mask

通过让Mask随时间演变，可以创建动态的力场效果： 

// VEX: 时间变化的球形Mask
float time = @Frame / 24.0;  // 转换为秒
vector center = chv("center");
float base_radius = chf("base_radius");
float expansion_rate = chf("expansion_rate");

// 随时间扩张的半径
float current_radius = base_radius + expansion_rate * time;
float dist = length(@P - center);

// 平滑的Mask边缘
float mask = 1.0 - smooth(0, current_radius, dist);

// 应用时间变化的力
vector impulse = chv("impulse_direction") * chf("impulse_strength");
@v += impulse * mask * @TimeInc;

5.3 相机投影策略（Camera Projection）

5.3.1 扩展Frustum方法

最实用的工程方案是扩展相机视野以包含完整运动轨迹，然后在引擎中补偿： 

计算步骤：

1. 预估最大位移
   d_max = max(|u|) × t_lifetime

2. 计算所需frustum扩展
   frustum_scale = 1 + (d_max / base_frustum_size) × safety_factor

3. 设置渲染相机
   - 保持原相机位置
   - 扩大ortho width / fov
   - 或使用非正方形aspect ratio

4. 引擎端补偿
   sprite_scale = 1 / frustum_scale

5.3.2 非正方形UV布局

对于水平或垂直方向性强的效果，使用非正方形帧可以最大化纹素利用率： 

// 水平方向效果：2:1 或 4:1 的长宽比
// 垂直方向效果：1:2 或 1:4 的长宽比

// Houdini Flipbook UV生成（sopcop2d或直接UV）
// 假设渲染分辨率为 1024x256 (4:1)

int frames = chi("num_frames");
int cols = chi("columns");  // 每行帧数

// 计算当前帧在sheet中的位置
int frame = @Frame - chi("start_frame");
int row = frame / cols;
int col = frame % cols;

// 计算UV偏移
float u_scale = 1.0 / cols;
float v_scale = 1.0 / ceil(float(frames) / cols);

@uv = set(
    (col + @uv[0]) * u_scale,
    1.0 - (row + @uv[1]) * v_scale  // V轴翻转
);

5.4 参考系转换（Reference Frame Transformation）

5.4.1 跟随相机参考系

在模拟空间中使用跟随相机参考系，使"流动"的粒子在相机坐标下保持相对静止：

设世界坐标系下的粒子位置为 xworld
相机参考系下的位置为 xcam
xcam = xworld - c(t)
其中 c(t) 为随时间移动的相机位置

这种方法需要修改速度场：

ucam = uworld - dc/dt
🎬 实际应用说明

跟随相机参考系在概念上优雅，但在Houdini中实现较为复杂。需要创建自定义的POP Solver或使用Python SOP来管理相机变换。实践中，扩展Frustum配合引擎缩放往往能以更小的工程成本达到相近的视觉效果。

6. 序列帧优化与导出

6.1 UV布局优化

6.1.1 Flipbook Sheet布局策略

将序列帧排列到单个纹理（Flipbook Sheet）时，布局策略直接影响纹素密度和采样效率。

布局类型 适用场景 优点 缺点
Sequential（顺序） 通用 直观、易调试 非2的幂时边缘浪费
Zigzag（锯齿） 长序列 更好的空间局部性 UV计算稍复杂
Non-square（非正方形） 定向效果 减少空白像素 需要自定义UV
RGBA Packing 灰度效果 4x纹素密度 仅适用于单通道

6.1.2 最优行列计算

给定总帧数N，计算最优的行列排列： 

// Python: 计算最优Flipbook布局
import math

def calculate_optimal_layout(num_frames, aspect_ratio=1.0):
    """
    计算最优的行列数
    aspect_ratio: 单帧的长宽比 (width/height)
    """
    best_waste = float('inf')
    best_cols, best_rows = 1, num_frames
    
    for cols in range(1, num_frames + 1):
        rows = math.ceil(num_frames / cols)
        
        # 计算最终纹理的长宽比
        sheet_width = cols * aspect_ratio
        sheet_height = rows
        sheet_ratio = sheet_width / sheet_height
        
        # 目标是接近正方形 (ratio = 1)
        waste = abs(sheet_ratio - 1.0)
        
        if waste < best_waste:
            best_waste = waste
            best_cols, best_rows = cols, rows
    
    return best_cols, best_rows

# 示例：64帧，2:1长宽比
cols, rows = calculate_optimal_layout(64, 2.0)
print(f"最优布局: {cols}列 × {rows}行")  # 输出: 8列 × 8行

6.2 RGBA通道打包

6.2.1 四帧打包原理

对于灰度效果（如水花、烟雾），可以将四帧序列帧打包到单个纹理的RGBA通道，实现4倍纹素密度： 

打包方式：
┌─────────────────┐
│  R  │  Frame 0  │
│  G  │  Frame 1  │
│  B  │  Frame 2  │
│  A  │  Frame 3  │
└─────────────────┘

渲染时的帧索引计算：
- packed_frame = floor(normalized_time * num_frames / 4)
- channel = packed_frame % 4
- sub_frame = floor(normalized_time * num_frames) % 4

6.2.2 Houdini COP2实现

// COP2 VEX: RGBA通道打包
// 输入: 4帧连续的灰度图像序列

int frame_offset = ch("frame_offset");
int pack_index = @frame - frame_offset;

// 确定当前层写入哪个通道
// pack_index 0-3 对应 RGBA
float value = @C.r;  // 假设输入是灰度

// 清空所有通道
@C = 0;

// 根据pack_index写入对应通道
if(pack_index == 0) @C.r = value;
else if(pack_index == 1) @C.g = value;
else if(pack_index == 2) @C.b = value;
else if(pack_index == 3) @C.a = value;

6.3 Motion Vectors生成

6.3.1 运动向量原理

Motion Vectors记录了像素在屏幕空间的位移，用于运行时运动模糊或流体扭曲效果：

MV = (Pscreen(t+1) - Pscreen(t)) · scale

6.3.2 Houdini生成实现

// VEX: 在着色器中生成Motion Vectors
// 需要前一帧和当前帧的velocity属性

// 计算屏幕空间位置
vector ndc_current = toNDC(@P);
vector ndc_next = toNDC(@P + @v * @TimeInc);

// 计算屏幕空间位移 (范围 -1 到 1)
vector2 screen_vel = set(
    ndc_next.x - ndc_current.x,
    ndc_next.y - ndc_current.y
);

// 编码到 0-1 范围输出
@mv = set(
    screen_vel.x * 0.5 + 0.5,
    screen_vel.y * 0.5 + 0.5,
    0
);

6.4.2 Unity VFX Graph设置

// VFX Graph: 在Output Context中设置
// 或在Initialize Particle中设置Size

// 如果使用Flipbook Shader Graph:
// 在Shader中处理UV缩放

void FlipbookUV_float(
    float2 UV,
    float Frames,
    float Columns,
    float Rows,
    float FrameIndex,
    float AspectScale,  // 扩展补偿参数
    out float2 OutUV
) {
    float col = fmod(FrameIndex, Columns);
    float row = floor(FrameIndex / Columns);
    
    float2 scale = float2(1.0/Columns, 1.0/Rows) * AspectScale;
    float2 offset = float2(col/Columns, 1.0 - (row+1)/Rows);
    
    OutUV = UV * scale + offset;
}

7.2 案例二：爆炸定向飞溅

本案例展示爆炸效果中向上飞溅的水花制作。

7.2.1 技术特点

  • 强垂直方向性（向上喷射）
  • 需要重力作用下的回落
  • 高度衰减的湍流细节

7.2.2 核心VEX代码

// POP Wrangle: 爆炸初始冲击
float explosion_strength = chf("explosion_strength");
float explosion_radius = chf("explosion_radius");

vector center = chv("explosion_center");
vector to_center = @P - center;
float dist = length(to_center);

// 径向衰减
float falloff = fit(dist, 0, explosion_radius, 1, 0);
falloff = pow(falloff, 2);

// 主要向上，带径向扩散
vector impulse = normalize(set(
    to_center.x * 0.3,
    1.0,  // 主要向上
    to_center.z * 0.3
));

@v += impulse * explosion_strength * falloff;

// 添加初始涡旋
vector vortex = cross(to_center, set(0,1,0));
@v += normalize(vortex) * explosion_strength * 0.2 * falloff;

# Python: 程序化创建FLIP水花模拟网络
import hou

def create_flip_splash_setup():
    """创建完整的Flipbook水花FLIP模拟设置"""
    
    obj = hou.node("/obj")
    
    # 创建Geometry节点
    geo = obj.createNode("geo", "flip_splash_setup")
    
    # 1. 创建发射器 (Sphere)
    sphere = geo.createNode("sphere")
    sphere.parm("type").set(1)  # Polygon mesh
    sphere.parm("radx").set(0.3)
    sphere.parm("rady").set(0.1)
    sphere.parm("radz").set(0.3)
    
    # 2. 创建粒子发射
    scatter = geo.createNode("scatter")
    scatter.setInput(0, sphere)
    scatter.parm("npts").set(5000)
    
    # 3. 添加速度属性（初始向上喷射）
    v_from_curves = geo.createNode("pointwrangle")
    v_from_curves.setInput(0, scatter)
    v_from_curves.parm("snippet").set("""
// 初始速度：向上带随机扩散
float spread = 0.5;
vector base_dir = set(0, 1, 0);
vector rand_dir = normalize(set(
    fit01(rand(@elemnum), -spread, spread),
    0,
    fit01(rand(@elemnum + 1), -spread, spread)
));

@v = normalize(base_dir + rand_dir) * 5.0;
""")
    
    # 4. 创建DOP Network
    dopnet = obj.createNode("dopnet", "flip_sim")
    
    # 5. 创建FLIP Object
    flipobj = dopnet.createNode("flipobject")
    flipobj.parm("soppath").set(v_from_curves.path())
    flipobj.parm("particlesep").set(0.05)
    
    # 6. 创建FLIP Solver
    flipsolver = dopnet.createNode("flipsolver")
    flipsolver.setInput(0, flipobj)
    flipsolver.parm("flipsolver").set(1)  # Use FLIP
    
    # 7. 添加重力
    gravity = dopnet.createNode("gravity")
    gravity.setInput(0, flipsolver, 1)
    flipsolver.setInput(1, gravity)
    
    # 8. 添加POP Wrangle（定向力）
    popwrangle = dopnet.createNode("popwrangle")
    popwrangle.setInput(0, flipsolver, 1)
    popwrangle.parm("snippet").set("""
// 定向力 + 边界约束
vector force = chv("force_direction") * chf("force_scale");
@v += force * @TimeInc;

// 软边界约束
vector bbox_min = chv("bbox_min");
vector bbox_max = chv("bbox_max");
float stiffness = chf("boundary_stiffness");

for(int i = 0; i < 3; i++) {
    if(@P[i] < bbox_min[i]) {
        @v[i] += stiffness * (bbox_min[i] - @P[i]) * @TimeInc;
    }
    if(@P[i] > bbox_max[i]) {
        @v[i] -= stiffness * (@P[i] - bbox_max[i]) * @TimeInc;
    }
}
""")
    flipsolver.setInput(1, popwrangle)
    
    # 9. 创建输出节点
    output = dopnet.createNode("output")
    output.setInput(0, flipsolver)
    
    dopnet.layoutChildren()
    geo.layoutChildren()
    
    return geo, dopnet

# 执行创建
geo, dopnet = create_flip_splash_setup()
print(f"Created: {geo.path()}, {dopnet.path()}")

# Python: 生成Flipbook UV布局并导出
import hou
import math

def create_flipbook_uv_and_export():
    """创建Flipbook UV布局并设置导出"""
    
    geo = hou.node("/obj/flip_splash_setup")
    
    # 1. 创建UV布局节点
    uvtransform = geo.createNode("uvtransform")
    
    # 2. 设置Flipbook参数
    num_frames = 64
    aspect_ratio = 4.0  # 水平方向效果
    
    # 计算最优布局
    best_waste = float('inf')
    best_cols, best_rows = 1, num_frames
    
    for cols in range(1, num_frames + 1):
        rows = math.ceil(num_frames / cols)
        sheet_ratio = (cols * aspect_ratio) / rows
        waste = abs(sheet_ratio - 1.0)
        if waste < best_waste:
            best_waste = waste
            best_cols, best_rows = cols, rows
    
    print(f"Optimal layout: {best_cols} x {best_rows}")
    
    # 3. 创建材质
    matbuilder = hou.node("/mat").createNode("principledshader")
    matbuilder.setName("flipbook_water_mat")
    matbuilder.parm("basecolorr").set(0.9)
    matbuilder.parm("basecolorg").set(0.95)
    matbuilder.parm("basecolorb").set(1.0)
    matbuilder.parm("roughness").set(0.2)
    matbuilder.parm("transparency").set(0.3)
    
    # 4. 创建渲染ROP
    ropnet = hou.node("/out")
    
    # Mantra渲染节点
    mantra = ropnet.createNode("ifd")
    mantra.setName("flipbook_render")
    mantra.parm("vm_renderengine").set(2)  # PBR
    mantra.parm("camera").set("/obj/cam1")
    mantra.parm("vm_picture").set("$HIP/render/flipbook_$F4.png")
    mantra.parm("f1").set(1)
    mantra.parm("f2").set(num_frames)
    
    # 5. 创建COP2合成网络（生成sheet）
    copnet = hou.node("/img").createNode("img", "flipbook_comp")
    
    # File节点读取序列
    file_node = copnet.createNode("file")
    file_node.parm("file").set("$HIP/render/flipbook_$F4.png")
    
    # Mosaic节点生成sheet
    mosaic = copnet.createNode("mosaic")
    mosaic.setInput(0, file_node)
    mosaic.parm("cols").set(best_cols)
    mosaic.parm("rows").set(best_rows)
    
    # ROP Output
    rop = copnet.createNode("rop_comp")
    rop.setInput(0, mosaic)
    rop.parm("copoutput").set("$HIP/export/water_splash_flipbook.png")
    rop.parm("trange").set(1)  # Render current frame only
    
    copnet.layoutChildren()
    
    return mantra, rop

# 执行
mantra, rop = create_flipbook_uv_and_export()

8.3 引擎端Shader (HLSL)

// HLSL: Unity URP Flipbook Shader with Directional Compensation
Shader "VFX/FlipbookWater"
{
    Properties
    {
        _MainTex ("Flipbook Texture", 2D) = "white" {}
        _MotionVectorTex ("Motion Vectors", 2D) = "black" {}
        
        [Header(Flipbook Settings)]
        _Frames ("Total Frames", Float) = 64
        _Columns ("Columns", Float) = 8
        _Rows ("Rows", Float) = 8
        _Speed ("Play Speed", Float) = 1.0
        _ScaleCompensation ("Scale Compensation", Float) = 1.0
        
        [Header(Visual Settings)]
        _Color ("Tint", Color) = (1,1,1,1)
        _Intensity ("Intensity", Float) = 1.0
        _Softness ("Edge Softness", Float) = 0.5
        _UseMotionVectors ("Use Motion Vectors", Float) = 0
    }
    
    SubShader
    {
        Tags { "RenderType"="Transparent" "Queue"="Transparent" }
        
        Pass
        {
            Name "FORWARD"
            Tags { "LightMode"="UniversalForward" }
            
            Blend SrcAlpha OneMinusSrcAlpha
            ZWrite Off
            Cull Off
            
            HLSLPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            #pragma multi_compile_fog
            
            #include "Packages/com.unity.render-pipelines.universal/ShaderLibrary/Core.hlsl"
            
            struct Attributes
            {
                float4 positionOS : POSITION;
                float2 uv : TEXCOORD0;
                float4 color : COLOR;
            };
            
            struct Varyings
            {
                float4 positionCS : SV_POSITION;
                float2 uv : TEXCOORD0;
                float4 color : COLOR;
                float fogFactor : TEXCOORD1;
            };
            
            TEXTURE2D(_MainTex);
            SAMPLER(sampler_MainTex);
            TEXTURE2D(_MotionVectorTex);
            SAMPLER(sampler_MotionVectorTex);
            
            CBUFFER_START(UnityPerMaterial)
                float4 _MainTex_ST;
                float4 _Color;
                float _Frames;
                float _Columns;
                float _Rows;
                float _Speed;
                float _ScaleCompensation;
                float _Intensity;
                float _Softness;
                float _UseMotionVectors;
            CBUFFER_END
            
            // 计算Flipbook UV
            float2 CalculateFlipbookUV(float2 uv, float normalizedTime)
            {
                float currentFrame = normalizedTime * _Frames;
                float frameIndex = floor(currentFrame);
                float frameFraction = frac(currentFrame);
                
                // 计算行列位置
                float col = fmod(frameIndex, _Columns);
                float row = floor(frameIndex / _Columns);
                
                // 计算单个frame的UV范围
                float2 frameSize = float2(1.0 / _Columns, 1.0 / _Rows);
                
                // 应用Scale补偿后的UV
                float2 localUV = uv / _ScaleCompensation;
                localUV = saturate(localUV);  // Clamp to 0-1
                
                // 计算最终UV
                float2 finalUV = localUV * frameSize;
                finalUV.x += col * frameSize.x;
                finalUV.y += (_Rows - 1 - row) * frameSize.y;  // Flip Y
                
                return finalUV;
            }
            
            Varyings vert(Attributes input)
            {
                Varyings output;
                
                VertexPositionInputs posInputs = GetVertexPositionInputs(input.positionOS.xyz);
                output.positionCS = posInputs.positionCS;
                output.uv = input.uv;
                output.color = input.color;
                output.fogFactor = ComputeFogFactor(posInputs.positionCS.z);
                
                return output;
            }
            
            half4 frag(Varyings input) : SV_Target
            {
                // 计算归一化时间 (0-1循环)
                float normalizedTime = frac(_Time.y * _Speed);
                
                // 计算Flipbook UV
                float2 flipbookUV = CalculateFlipbookUV(input.uv, normalizedTime);
                
                // 采样主纹理
                half4 col = SAMPLE_TEXTURE2D(_MainTex, sampler_MainTex, flipbookUV);
                
                // 应用颜色、强度和顶点色
                col *= _Color * _Intensity * input.color;
                
                // 边缘软化
                float2 centerDist = abs(input.uv - 0.5) * 2.0;
                float edgeMask = 1.0 - saturate(max(centerDist.x, centerDist.y) - (1.0 - _Softness));
                col.a *= edgeMask;
                
                // 应用雾效
                col.rgb = MixFog(col.rgb, input.fogFactor);
                
                return col;
            }
            ENDHLSL
        }
    }
}
