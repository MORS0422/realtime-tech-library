Niagara Decal 渲染器完全指南：投影映射、Deferred Decal 与动态淡入淡出

Decal（贴花）渲染是实时图形学中在现有几何表面上叠加细节的关键技术。从弹孔、脚印到魔法符文， Decal 系统为场景增添了丰富的视觉细节。Niagara Decal 渲染器将粒子系统的动态性与贴花投影相结合， 实现了完全基于 GPU 的实时贴花生成与管理。本文从投影几何学出发，深入解析 Projective Mapping 数学原理、 Deferred Decal 管线架构和动态淡入淡出算法。

📋 内容摘要

Decal 渲染的本质是投影映射（Projective Mapping）——将 2D 纹理通过几何变换投影到 3D 场景表面。 本文系统讲解：Box Projection 投影几何（正交/透视投影矩阵与裁剪空间变换）、 Deferred Decal 渲染管线（GBuffer 与 DBuffer 两种架构的数学原理与性能权衡）、 Niagara Decal 渲染器（GPU 粒子驱动的动态贴花系统）、 以及 淡入淡出算法（基于生命周期与距离场的透明度混合）。 通过符号定义、投影推导和 Niagara 材质实现，掌握现代实时贴花技术。

1. 概述与核心概念

1.1 什么是 Projective Mapping？

Projective Mapping（投影映射）是将纹理坐标通过投影变换映射到 3D 几何表面的技术。 与 UV 映射不同，投影映射不需要预先为网格展开 UV，而是通过实时计算的几何投影确定纹理采样位置。 这使得 Decal 可以"附着"在任意复杂几何上，适应曲率和表面细节。

🎯 核心洞察

Decal 投影的核心数学工具是投影矩阵（Projection Matrix）。 通过定义一个投影体积（通常是 Box 或 Frustum），我们将世界空间坐标变换到投影空间的 UV 坐标。 这种变换是可逆的：给定世界坐标，可以计算其在 Decal 纹理上的对应位置； 反之，给定 Decal UV，可以重构世界空间射线进行相交测试。 正是这一数学特性使得 Decal 能够动态适应任意几何表面。

1.2 Decal 渲染技术演进

发展阶段 核心技术 局限性
前向贴花 (2000s) 几何体挤出、Alpha 混合 Z-fighting、深度冲突

3. 投影映射核心理论

3.1 Box Projection 数学原理

Box Projection（盒式投影）是 Decal 渲染的标准方法。Decal 被定义为一个有方向的 Box， 纹理沿着 Box 的主轴（通常是 Z 轴）正交投影到场景几何上。 

Box Projection 坐标变换：

1. 世界空间到局部空间：
   Plocal = Mdecal-1 × [x, y, z, 1]T

2. 局部空间到 UV 空间：
   u = (Plocal.x / Sx + 1) / 2
   v = (Plocal.y / Sy + 1) / 2
   d = (Plocal.z / Sz + 1) / 2

3. 裁剪测试：
   if (u < 0 || u > 1 || v < 0 || v > 1 || d < 0 || d > 1)
       discard;  // 在 Decal Box 外部

4. 纹理采样：
   Color = Texture2D(DecalTexture, float2(u, v))

其中：
- Mdecal = T × R × S（平移 × 旋转 × 缩放矩阵）
- S = (Sx, Sy, Sz) 是 Box 的半尺寸
- (u,v) 是归一化的纹理坐标
- d 是归一化的投影深度
💡 数学洞察

Box Projection 的关键在于Mdecal-1 的计算。 由于 Mdecal 是正交矩阵（仅包含旋转和平移），其逆矩阵等于转置矩阵加上平移修正： Mdecal-1 = [RT | -RT×T]。 这一性质使得 GPU 上的变换计算非常高效。 此外，Box Projection 产生的 UV 坐标在投影方向上保持线性， 这确保了纹理在平面表面上不会产生透视扭曲。

3.2 深度测试与表面贴合

Decal 必须与场景几何正确贴合。这需要比较 Decal 投影深度与场景深度缓冲， 确保 Decal 只出现在与投影射线相交的表面上。 

深度贴合算法：

1. 屏幕空间 Decal 位置计算：
   Pscreen = ViewProjMatrix × Pworld
   UVscreen = Pscreen.xy / Pscreen.w × 0.5 + 0.5

2. 采样场景深度：
   Depthscene = SampleSceneDepth(UVscreen)

3. 重构场景世界位置：
   Pscene = ReconstructWorldPosition(UVscreen, Depthscene)

4. 计算 Decal UV：
   Plocal = Mdecal-1 × Pscene
   (u, v, d) = LocalToUV(Plocal, S)

5. 深度差异测试：
   Δd = |d - 0.5|  // 相对于 Decal Box 中心的距离
   if (Δd > 0.5) discard;  // 超出 Decal Box 深度范围

6. 深度柔和：
   fdepth = 1 - smoothstep(0, fdepthRange, Δd)

其中 fdepthRange 控制深度边缘的柔和程度，通常设为 0.1-0.2。

3.3 Normal Blending 原理

Decal 不仅要影响 Base Color，还需要影响表面法线以产生真实的凹凸效果。 Normal Blending（法线混合）是将 Decal 的切线空间法线与表面法线正确组合的过程。 

法线混合数学：

1. 解码 Decal 法线（从切线空间）：
   NdecalTS = DecodeNormal(DecalNormalTexture(u,v))
   NdecalWS = TBNdecal × NdecalTS

2. 获取表面法线（来自 GBuffer）：
   Nsurface = GBuffer.Normal

3. 归一化混合（Reoriented Normal Mapping）：
   Nblended = normalize(Nsurface + NdecalWS × βnormal × α)

4. 高级混合（UDN Blending）：
   t = NdecalTS × 2 - 1
   u = Nsurface × 2 - 1
   r = normalize(float3(t.xy + u.xy, t.z × u.z))
   Nblended = r × 0.5 + 0.5

5. 混合权重控制：
   βnormal = DecalNormalIntensity × fedge × fdepth × α

其中：
- TBNdecal 是 Decal 的切线-副切线-法线矩阵
- βnormal 是法线混合强度
- α 是 Decal 透明度

4. Decal 渲染管线架构

4.1 延迟渲染管线中的 Decal

Deferred Decal 管线：

场景渲染阶段
├─ GBuffer Pass
│  ├─ Base Color → GBufferA
│  ├─ Normal      → GBufferB
│  ├─ ORM         → GBufferC
│  └─ Depth       → ZBuffer
│
Decal 渲染阶段
├─ 遍历所有 Decal 体积
├─ 对于每个 Decal:
│  ├─ 计算投影矩阵 Mdecal
│  ├─ 绑定 Decal 材质
│  └─ 全屏/体积 Pass:
│     ├─ 采样 GBuffer 深度
│     ├─ 重构世界位置
│     ├─ 计算 Decal UV
│     ├─ 采样 Decal 纹理
│     └─ 混合到 GBuffer
│
光照阶段
├─ 使用修改后的 GBuffer 进行光照计算
└─ 输出最终颜色

混合公式（GBuffer Decal）：
GBuffer′ = GBuffer × (1 - α) + Decal × α

其中 α 是 Decal 在对应像素上的透明度。

4.2 DBuffer Decal 架构

DBuffer（Decal Buffer）是 UE4/5 引入的 Decal 架构改进。 它在 Base Pass 之前应用 Decal，将结果写入独立的 DBuffer 纹理， 然后在 Base Pass 中合并。这种方法支持前向渲染物体和静态光照。 

DBuffer 管线：

Pre-Base Pass 阶段
├─ Decal 投影 Pass
│  ├─ 遍历 Decal 体积
│  ├─ 计算 DBuffer 贡献
│  └─ 写入 DBuffer
│     ├─ DBufferA: BaseColor × Alpha
│     ├─ DBufferB: Normal × Alpha
│     └─ DBufferC: ORM × Alpha
│
Base Pass 阶段
├─ 对于每个像素:
│  ├─ 采样 DBuffer
│  ├─ 材质属性 = lerp(材质, DBuffer, DBuffer.Alpha)
│  └─ 应用光照
│
优势：
- 支持前向渲染物体
- 支持静态光照（烘焙光照）
- Decal 可以影响光照计算

劣势：
- 需要额外的内存带宽（DBuffer 写入/读取）
- 每个像素最多支持有限数量的 Decal（通常 3-4 个）

4.3 架构对比

特性 Deferred Decal DBuffer Decal
应用时机 GBuffer 之后 Base Pass 之前
前向渲染支持 不支持 支持
静态光照 不支持 支持
内存带宽 低 中（额外 DBuffer）
每像素 Decal 数 无限制 有限（通常 3-4）
性能 更高（Overdraw 可控） 中等（额外 Pass）

5. Niagara Decal 渲染器实现

5.1 Niagara 系统架构

Niagara Decal 数据流：

粒子生成阶段
├─ CPU Emitter:
│  ├─ 碰撞事件触发 Spawn
│  ├─ 设置初始属性:
│  │  ├─ Position: 碰撞点
│  │  ├─ Rotation: 对齐表面法线
│  │  ├─ Size: Decal 尺寸
│  │  ├─ Color: 颜色/透明度
│  │  ├─ Lifetime: Tlife
│  │  ├─ FadeIn: Tin
│  │  └─ FadeOut: Tout
│  └─ 生成 GPU 粒子
│
GPU 模拟阶段
├─ Normalize Age:
│  └─ τ = Age / Lifetime
│
├─ 淡入淡出计算:
│  ├─ 归一化淡入: τin = saturate(Age / FadeIn)
│  ├─ 归一化淡出: τout = 1 - saturate((Age - (Lifetime - FadeOut)) / FadeOut)
│  └─ α(τ) = τin × τout
│
├─ 动态属性更新:
│  ├─ 可选: 尺寸动画
│  ├─ 可选: 颜色变化
│  └─ 可选: 位置漂移
│
Decal 渲染阶段
├─ 每粒子渲染:
│  ├─ 构建 Mdecal 矩阵
│  ├─ 投影到 GBuffer/DBuffer
│  └─ 应用材质与透明度
│
数据传递:
Particle Attributes → Material Parameter → Decal Shader

5.2 淡入淡出曲线系统

淡入淡出是 Decal 生命周期管理的核心。通过控制透明度随时间的变化， 可以实现自然出现和消失的效果。 

透明度曲线数学：

1. 线性淡入淡出：
   ⎧ t/Tin                          当 0 ≤ t < Tin
α(t) = ⎨ 1                              当 Tin ≤ t < Tlife - Tout
   ⎩ 1 - (t - (Tlife - Tout))/Tout   当 Tlife - Tout ≤ t ≤ Tlife

2. 平滑淡入淡出（smoothstep）：
   αin(t) = 3(t/Tin)² - 2(t/Tin)³
   αout(t) = 1 - [3(τout)² - 2(τout)³]
   其中 τout = (t - (Tlife - Tout))/Tout

3. 指数淡入淡出：
   αin(t) = 1 - e-kt/Tin
   αout(t) = e-k(t - (Tlife - Tout))/Tout

4. 自定义曲线（Niagara Float Curve）：
   α(t) = SampleCurve(NormalizedAge)

Niagara 模块参数：
- Fade In Duration: Tin
- Fade Out Duration: Tout
- Fade In Curve: 曲线类型（Linear/Smooth/Exponential/Custom）
- Fade Out Curve: 同上

5.3 HLSL 材质实现

// Niagara Decal 材质 - DBuffer 输出
// 输入：Niagara 粒子属性

// 材质参数（由 Niagara 驱动）
float DecalAlpha;           // 粒子透明度 α(t)
float3 DecalColor;          // 粒子颜色
float NormalIntensity;      // 法线混合强度
float EdgeSoftness;         // 边缘柔和范围
float DepthSoftness;        // 深度柔和范围

// Decal 投影函数
float3 ComputeDecalUV(float3 WorldPosition, float4x4 DecalInvMatrix, float3 DecalSize)
{
    // 变换到 Decal 局部空间
    float3 LocalPos = mul(DecalInvMatrix, float4(WorldPosition, 1.0)).xyz;
    
    // 归一化到 [0,1] 范围
    float3 UVW = LocalPos / DecalSize * 0.5 + 0.5;
    
    return UVW;
}

// 边缘柔和函数
float ComputeEdgeFade(float2 UV, float Softness)
{
    float2 EdgeDist = min(UV, 1.0 - UV);
    float MinEdge = min(EdgeDist.x, EdgeDist.y);
    return smoothstep(0, Softness, MinEdge);
}

// 深度柔和函数
float ComputeDepthFade(float Depth, float Softness)
{
    // Depth 在 [0,1] 范围内，0.5 是中心
    float DistFromCenter = abs(Depth - 0.5) * 2.0;
    return 1.0 - smoothstep(1.0 - Softness, 1.0, DistFromCenter);
}

// 主像素着色器
void DecalPixelShader(
    float2 ScreenUV : TEXCOORD0,
    float3 WorldPosition : TEXCOORD1,
    out float4 DBufferA : SV_Target0,  // BaseColor
    out float4 DBufferB : SV_Target1,  // Normal
    out float4 DBufferC : SV_Target2   // ORM
)
{
    // 计算 Decal UV
    float3 UVW = ComputeDecalUV(WorldPosition, DecalInvMatrix, DecalSize);
    
    // 裁剪测试
    if (any(UVW < 0.0) || any(UVW > 1.0))
        discard;
    
    // 采样 Decal 纹理
    float4 DecalBase = DecalBaseTexture.Sample(UVW.xy);
    float4 DecalNormal = DecalNormalTexture.Sample(UVW.xy);
    float4 DecalORM = DecalORMTexture.Sample(UVW.xy);
    
    // 计算柔和因子
    float EdgeFade = ComputeEdgeFade(UVW.xy, EdgeSoftness);
    float DepthFade = ComputeDepthFade(UVW.z, DepthSoftness);
    
    // 最终透明度
    float FinalAlpha = DecalBase.a * DecalAlpha * EdgeFade * DepthFade;
    
    // 应用颜色
    float3 FinalColor = DecalBase.rgb * DecalColor;
    
    // 解码并混合法线
    float3 DecalNormalTS = DecodeNormal(DecalNormal);
    float3 DecalNormalWS = mul(DecalNormalTS, DecalTBNMatrix);
    
    // DBuffer 输出（预乘 Alpha）
    DBufferA = float4(FinalColor * FinalAlpha, FinalAlpha);
    DBufferB = float4(DecalNormalWS * 0.5 + 0.5, FinalAlpha) * FinalAlpha;
    DBufferC = float4(DecalORM.rgb, FinalAlpha) * FinalAlpha;
}

6. 高级技术与优化

6.1 距离场 Decal（Mesh Distance Field）

UE5 引入了基于网格距离场（Mesh Distance Field, MDF）的 Decal 技术。 这种方法无需投影矩阵，直接利用距离场查询表面位置和法线。 

距离场 Decal 原理：

1. 距离场定义：
   D(p) = 点 p 到最近表面的有符号距离
   - D < 0: 在物体内部
   - D > 0: 在物体外部
   - D = 0: 在表面上

2. 表面重构：
   给定距离场 D(p)，表面法线可以通过梯度计算：
   N = ∇D = (dD/dx, dD/dy, dD/dz)

3. Decal 应用：
   对于 Decal 中心点 C 和半径 R：
   
   for 每个像素在球体 C,R 内:
       p = 射线与距离场表面交点
       if D(p) ≈ 0:
           UV = ComputeUVFromPosition(p)
           ApplyDecal(UV)

4. 优势：
   - 无需投影矩阵
   - 自动适应任意曲面
   - 支持体积贴花（Volume Decal）

5. 劣势：
   - 需要预计算距离场（内存开销）
   - 仅支持静态网格
   - 精度受距离场分辨率限制

6.2 体积贴花 vs 表面贴花

特性 表面贴花（Surface Decal） 体积贴花（Volume Decal）
投影方式 沿单一方向投影 包裹整个体积内的表面
适用表面 朝向投影方向的表面 体积内所有方向的表面
法线处理 基于投影方向 基于表面实际法线
应用场景 弹孔、脚印、贴地标记 涂鸦、伤痕、腐蚀效果
性能 更高（单一方向测试） 较低（多方向测试）
距离场支持 可选 推荐

6.3 性能优化策略

Niagara Decal 性能指南

  • • Decal 数量：同屏活跃 Decal 控制在 50-100 个以内
  • • 尺寸限制：单个 Decal 尺寸不宜超过 2-3 米
  • • 生命周期：设置合理的 FadeOut 时间，及时回收
  • • LOD 系统：远距离降低贴花分辨率或禁用
  • • 材质优化：使用简单的 Decal 材质，避免复杂计算
  • • 视图剔除：利用 Niagara 的视图剔除功能

7. 实战应用场景

7.1 典型应用案例

游戏内应用：

1. 弹孔与弹痕
   - 触发：武器射线检测碰撞
   - 设置：对齐碰撞法线，随机旋转
   - 淡出：5-10 秒后淡出消失

2. 脚印系统
   - 触发：角色动画事件（脚步落地）
   - 设置：对齐地面法线，根据材质选择贴花
   - 淡出：根据材质（水泥保留久，雪地保留短）

3. 魔法符文
   - 触发：法术施放
   - 设置：发光贴花，颜色随法术类型变化
   - 动画：UV 流动、颜色脉动

4. 液体污渍
   - 触发：液体碰撞、角色死亡
   - 设置：次表面散射材质
   - 扩散：尺寸随时间轻微增大
